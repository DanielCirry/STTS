<objective>
Create FallbackAIManager backend and wire it into engine.py, replacing direct AIAssistantManager.generate() calls with automatic provider fallback.

Purpose: AI assistant currently has zero fallback -- when the active provider fails, the error surfaces raw to the user and conversation stops. This plan adds the orchestration layer that tries providers in priority order (local -> Groq -> Gemini -> OpenAI -> Anthropic), handles rate limits, timeouts, and network errors, and broadcasts provider switch events to the frontend.

Output: python/ai/assistant/fallback.py (new), modified engine.py (3 call sites), modified events.py (3 new event types)
</objective>

  <name>Task 1: Create FallbackAIManager with provider health tracking and fallback logic</name>
  <files>python/ai/assistant/fallback.py, python/ai/assistant/__init__.py</files>
  <action>
Create `python/ai/assistant/fallback.py` containing:

1. **ProviderHealth enum**: HEALTHY, COOLING (temporary RPM), EXHAUSTED (daily limit, skip for session).

2. **ProviderState dataclass**: `health`, `cooldown_until` (monotonic), `failure_count`, `last_error`. Methods:
   - `is_available()`: EXHAUSTED -> False; COOLING -> check `time.monotonic() >= cooldown_until` and auto-recover to HEALTHY; else True
   - `mark_rate_limited(retry_after, daily)`: daily sets EXHAUSTED, else COOLING with `cooldown_until = monotonic + max(retry_after, 5.0)`
   - `mark_failure(retry_after=30.0)`: COOLING with cooldown
   - `mark_healthy()`: reset health, failure_count, last_error

3. **PROVIDER_CONTEXT_LIMITS dict**: `{'local': 4, 'groq': 10, 'google': 10, 'openai': 10, 'anthropic': 10}`. Local gets 4 messages (2 turns) because local models are slow and have small context. Cloud gets full 10 (the existing `context_messages` default).

4. **FallbackAIManager class**:
   - `DEFAULT_PRIORITY = ['local', 'groq', 'google', 'openai', 'anthropic']`
   - `CLOUD_TIMEOUT = 15.0` seconds
   - `NETWORK_RETRY_INTERVAL = 60.0` seconds
   - `__init__(self, manager, notify_callback=None)`: stores ref to AIAssistantManager, async notify callback, health dict, priority list, active_provider, shared_conversation list, network_down flag, network_retry_at
   - `clear_conversation()`: clears shared_conversation only, NOT health states (per CONTEXT.md locked decision)
   - `get_active_provider()`: returns current active provider ID
   - `_state(provider_id)`: lazy-init ProviderState in health dict
   - `_get_candidates()`: iterate priority list, check `get_available_providers()` from manager. For 'local': check `is_loaded` (NOT `is_available` -- is_available only means llama_cpp installed). For cloud: check `is_available` (has API key). Skip cloud when `_network_down and monotonic < _network_retry_at`. Check `_state(pid).is_available()` for health. Return list of candidate IDs.
   - `_emit(event_type, data)`: async, calls notify_callback with try/except for safety
   - `generate(prompt) -> AssistantResponse`: The core fallback loop:
     a. Get candidates from `_get_candidates()`
     b. Apply sticky preference: if `_active_provider` in candidates, move it to front
     c. If no candidates: emit `ai_offline_mode`, return soft-failure `AssistantResponse(content="[AI unavailable] ...", tokens_used=0, model='fallback', truncated=False)`
     d. Track `network_fail_count = 0`
     e. For each candidate:
        - Get `provider_obj` from `self._manager._providers.get(provider_id)`. Call provider_obj.generate() DIRECTLY -- do NOT call manager.generate() to avoid on_response/on_error callbacks firing prematurely during fallback.
        - Inject conversation: `provider_obj._conversation = list(self._shared_conversation[-limit:])` where limit = PROVIDER_CONTEXT_LIMITS[provider_id]
        - For cloud providers: wrap with `asyncio.wait_for(provider_obj.generate(prompt), timeout=15.0)`. For local: call without timeout.
        - On SUCCESS: read back `provider_obj._conversation` into `_shared_conversation`. Mark healthy. Emit `ai_provider_switched` if provider changed (or first call with from=None). If was network_down, clear it and emit `ai_online_restored`. Return response.
        - On `asyncio.TimeoutError` (MUST catch BEFORE Exception): mark_failure(60.0), log warning, emit ai_provider_switched to next candidate if any, continue loop.
        - On `Exception`: call `_is_rate_limit(exc)` -- if True, parse retry_after header, check if daily, mark_rate_limited, emit switch, continue. Call `_is_network_error(exc)` -- if True, mark_failure, increment network_fail_count, if >= 2 and not already network_down then set _network_down=True and _network_retry_at and emit ai_offline_mode, continue. Else (application error): log error and RE-RAISE (do not swallow -- per CONTEXT.md locked decision, application errors surface to user).
     f. After loop exhausted: return soft-failure AssistantResponse with model='fallback'

5. **Module-level helper functions**:
   - `_is_rate_limit(exc)`: check `getattr(exc, 'status_code', None) in (429, 529)`, then try importing `google.api_core.exceptions.ResourceExhausted` isinstance check, then `anthropic.OverloadedError` isinstance check, then string fallback ('429', 'rate limit', 'resource_exhausted' in str(exc).lower())
   - `_is_network_error(exc)`: check isinstance (ConnectionError, TimeoutError, OSError), then google ServiceUnavailable/DeadlineExceeded, then type name string matching ('ConnectionError', 'Timeout', 'NetworkError', 'ServiceUnavailable', 'ConnectError')
   - `_is_daily_limit(exc)`: check 'per day', 'daily', 'quota' in str(exc).lower()
   - `_get_retry_after(exc, default=60.0)`: check exc.response.headers for 'retry-after', 'x-ratelimit-reset-requests', 'x-ratelimit-reset-tokens'. Parse as float, return default on failure.

6. **Update `python/ai/assistant/__init__.py`**: Add `FallbackAIManager` to exports.

Use the production-ready skeleton from `03-RESEARCH.md` as the primary reference. The code examples there have been verified against the actual source files.

IMPORTANT anti-patterns to avoid:
- Do NOT call `self._manager.generate()` inside FallbackAIManager -- this triggers on_response/on_error callbacks prematurely. Call `provider_obj.generate(prompt)` directly.
- Do NOT alias conversation lists -- always `list(...)` when injecting.
- Do NOT apply timeout to local LLM -- only cloud providers get 15s timeout.
- Catch `asyncio.TimeoutError` BEFORE `Exception` in the try/except chain.
