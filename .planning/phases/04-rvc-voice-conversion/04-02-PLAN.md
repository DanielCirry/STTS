    Modify `TTSManager` to include an optional RVC post-processing step between synthesis and playback.

    **Add to __init__:**
    ```python
    from ai.tts.rvc_postprocess import RVCPostProcessor

    # In __init__:
    self._rvc: Optional[RVCPostProcessor] = None
    self._on_rvc_failed: Optional[Callable] = None
    ```

    **Modify speak() method** — insert RVC processing between synthesis and playback:
    ```python
    async def speak(self, text: str) -> bool:
        # ... existing code up to result = await engine.synthesize(text) ...

        result = await engine.synthesize(text)

        if self._stop_requested:
            return False

        # RVC post-processing (optional)
        if self._rvc and self._rvc.is_enabled():
            try:
                result = await self._rvc.process(result)
            except Exception as e:
                logger.warning(f"RVC processing failed, using original audio: {e}")
                # Notify frontend via callback so engine.py can broadcast the event
                if self._on_rvc_failed:
                    self._on_rvc_failed(str(e))
                # Fall through to play original audio (fail-safe per user decision)

        # Play audio
        await self._play_audio(result)
        return True
    ```

    **Add RVC management methods to TTSManager:**

    1. `def get_rvc(self) -> Optional[RVCPostProcessor]:` — returns self._rvc (for engine.py to call methods on it directly)

    2. `def init_rvc(self) -> RVCPostProcessor:` — lazily initialize RVCPostProcessor if not yet created. Return the instance. This avoids importing torch and loading RVC code until the user actually wants RVC.
       ```python
       def init_rvc(self) -> RVCPostProcessor:
           if self._rvc is None:
               self._rvc = RVCPostProcessor()
           return self._rvc
       ```

    3. `def set_on_rvc_failed(self, callback: Optional[Callable[[str], None]]):` — sets the `_on_rvc_failed` callback. Engine.py will call this to register a callback that broadcasts the `rvc_conversion_failed` event.

    4. **Add RVC to cleanup():** If self._rvc is not None, call self._rvc.unload().

    **Important:** The RVC post-processor is lazily initialized. It is NOT created in __init__ to avoid loading torch/RVC code for users who never enable voice conversion. Only init_rvc() creates it.

    **Audio scope handling:** Per user decision, RVC should be configurable per audio source. For now in TTSManager, RVC only applies to TTS output (the default). The engine.py will handle the "which sources" logic if needed in the future — TTSManager just applies RVC if enabled.

    def _wire_rvc_failed_callback(self):
        """Wire TTSManager's on_rvc_failed to broadcast rvc_conversion_failed event."""
