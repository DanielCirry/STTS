  <action>
    Create `python/ai/tts/rvc_postprocess.py` with the `RVCPostProcessor` class that wraps the ported RVC inference code with a clean async interface.

    **Class design:**
    ```python
    class RVCPostProcessor:
        def __init__(self):
            self._enabled: bool = False
            self._model_path: Optional[str] = None
            self._index_path: Optional[str] = None
            self._device = None  # Set from ai.rvc.config

            # Loaded model state
            self._hubert: Optional[...] = None
            self._rmvpe: Optional[...] = None
            self._pipeline: Optional[Pipeline] = None
            self._loaded_model_name: Optional[str] = None

            # Parameters (with sensible defaults)
            self.f0_up_key: int = 0           # Semitone shift (-12 to +12)
            self.index_rate: float = 0.75     # FAISS influence (0.0-1.0)
            self.filter_radius: int = 3       # Pitch smoothing (1-7)
            self.rms_mix_rate: float = 0.25   # Loudness matching (0.0-1.0)
            self.protect: float = 0.33        # Consonant protection (0.0-0.5)
            self.resample_sr: int = 0         # 0 = no resample
            self.volume_envelope: float = 0.0 # Volume envelope mix

            # Progress callback for model loading
            self.on_progress: Optional[Callable[[str, float], None]] = None
            # Status callback for state changes
            self.on_status: Optional[Callable[[str, dict], None]] = None
    ```

    **Key methods:**

    1. `async def load_model(self, model_path: str, index_path: Optional[str] = None) -> bool:`
       - Validate .pth file exists
       - Auto-detect .index file if index_path is None (look for same-name .index in same directory)
       - Load HuBERT base model (download if not found — see below)
       - Load RMVPE model (download if not found)
       - Load voice .pth model (detect v1 vs v2 from config in checkpoint)
       - Load FAISS index if available (skip gracefully if faiss-cpu not installed)
       - All loading runs in `asyncio.get_event_loop().run_in_executor(None, sync_fn)` to avoid blocking
       - Report progress via on_progress callback: ("loading_hubert", 0.2), ("loading_rmvpe", 0.4), ("loading_voice", 0.6), ("loading_index", 0.8), ("ready", 1.0)
       - Return True on success, False on failure

    2. `async def process(self, result: TTSResult) -> TTSResult:`
       - Decode TTSResult.audio_data to float32 numpy array (handle WAV, MP3, raw PCM)
       - Resample to 16kHz mono for HuBERT input
       - Call pipeline.vc_single() in executor (CPU-bound, 1-5 seconds)
       - Encode output back to WAV bytes at the synthesizer's output sample rate
       - Return new TTSResult with converted audio
       - On ANY exception: log error, return original TTSResult unchanged (fail-safe per user decision)

    3. `def unload(self):`
       - Set _pipeline, _hubert, _rmvpe to None
       - Call `torch.cuda.empty_cache()` if CUDA available, or just `gc.collect()`
       - Reset _enabled to False
       - Report status via on_status callback

    4. `def is_enabled(self) -> bool:` — returns self._enabled and self._pipeline is not None

    5. `def enable(self, enabled: bool):` — sets self._enabled. If disabling, also call unload() per user decision.

    6. `def get_status(self) -> dict:` — returns current state (enabled, loaded, model_name, memory estimate)

    7. `def set_params(self, **kwargs):` — update f0_up_key, index_rate, filter_radius, etc.

    8. `def scan_models(self, directory: str) -> list:` — scan a directory for .pth files, return list of {name, path, size, modified} sorted by modified date descending, limited to most recent entries.

    **Base model download logic (HuBERT + RMVPE):**

    Default model directory: `models/rvc/pretrained/` relative to the STTS python root.

    Check order for HuBERT (hubert_base.pt):
    1. `models/rvc/pretrained/hubert_base.pt` (STTS local)
    2. Download from `https://huggingface.co/lj1995/VoiceConversionWebUI/resolve/main/hubert_base.pt`

    Same pattern for RMVPE (rmvpe.pt):
    1. `models/rvc/pretrained/rmvpe.pt` (STTS local)
    2. Download from `https://huggingface.co/lj1995/VoiceConversionWebUI/resolve/main/rmvpe.pt`

    Downloads should use `aiohttp` (already in requirements) or `requests` with progress tracking. Report download progress via on_progress callback so the frontend can show a progress bar. The download should NOT start automatically — the caller (engine.py in Plan 02) will prompt the user first per locked decision.

    **Audio format handling in process():**

    Different TTS engines return different formats:
    - Edge TTS: MP3 (24kHz) — detect by ID3/0xff header, decode with pydub
    - Piper: WAV (22050 Hz) — detect by RIFF header
    - SAPI: WAV (22050 Hz) — detect by RIFF header
    - VOICEVOX: WAV (24000 Hz) — detect by RIFF header

    After RVC conversion, output is always at the synthesizer's native rate (40000 Hz for v1, 32000 Hz for v2). Encode as WAV bytes in the returned TTSResult. The TTSManager._play_audio() already handles WAV decoding.

    **Thread safety:** All CPU-intensive operations (model loading, inference) MUST run in `asyncio.get_event_loop().run_in_executor(None, ...)`. Use a threading Lock for model load/unload to prevent concurrent access.
